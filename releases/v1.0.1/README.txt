WaterTempControl v1.0.1 二进制发布包

一、版本说明（Description）
本发布面向 STM32F103 水温控制固件的生产部署与现场调试，提供可直接烧录
的二进制镜像以及调试分析文件。v1.0.1 的核心目标是：

1) 在不改变既有控制策略与业务流程的前提下，提升输入链路的健壮性。
2) 在协议解析与参数写入路径上降低异常输入导致的配置污染风险。
3) 统一主机测试与目标构建的行为可见性，减少“仅某一构建路径复现”的问题。
4) 输出完整的可交付物（bin/hex/axf/map）与完整性校验信息，便于发布、追溯
	与验收。

本发布适用于以下典型场景：
- 量产烧录：使用 .bin 或 .hex 文件快速烧录。
- 联调排障：使用 .axf + .map 分析函数、符号与空间占用。
- 版本验收：使用 SHA256SUMS.txt 做文件一致性与防篡改校验。


二、软件功能详细说明（Software Functional Description）

2.1 系统功能总览
系统围绕“采集温度 -> 计算控制量 -> 执行加热 -> 人机交互/协议交互 -> 记录日志”
闭环运行，采用协作式调度机制，核心功能包括：

1) 温度采集与状态判定
- 读取 DS18B20 温度数据。
- 生成控制温度、有效位掩码、降级状态与故障状态。
- 对异常采样执行容错与状态上报。

2) 加热控制
- 根据设定温度、当前温度与 PID 参数计算输出。
- 驱动继电器执行加热开关控制。
- 在异常工况下优先执行安全策略（如关闭加热、报警）。

3) 报警与保护
- 支持阈值报警配置。
- 温度异常或故障条件下触发报警动作。
- 报警行为与控制行为解耦，便于维护与扩展。

4) 定时与计划任务
- 支持按分钟粒度的时段控制（启用/起始/结束）。
- 结合 RTC 时间实现计划生效与失效。

5) 参数管理与持久化
- 支持运行参数读写（设定温度、PID、报警阈值、日志周期、计划配置等）。
- 通过非易失存储保存参数，掉电后保持配置。

6) 人机界面与协议接口
- OLED 显示当前状态、参数与操作提示。
- 按键用于页面切换、参数调整、复位操作。
- UART 文本协议支持参数读取、参数设置、日志导出与清空。

7) 日志记录与导出
- 周期记录关键运行量（温度、设定值、PID 输出、加热状态、报警状态）。
- 支持文本导出，便于上位机采集与离线分析。


2.2 本版本重点功能改进

1) 协议输入校验加强
- 对配置命令参数采用严格解析，要求格式完整且可精确转换。
- 拒绝尾随多余字段，避免“部分解析成功但语义错误”的输入被接受。
- 拒绝非有限数值（NaN/Inf），防止异常浮点值进入控制链路。
- 对无效命令或无效负载返回明确错误响应，便于定位问题。

2) 默认值与局部变量安全性增强
- 协议处理路径中的局部变量显式初始化为默认值。
- 降低因未初始化读导致的不确定行为风险。

3) 主机/目标构建一致性修正
- DS18B20 诊断相关状态在主机测试路径与目标路径中可见性一致。
- 降低“测试通过但目标异常”或“目标正常但测试失败”的概率。


2.3 关键协议行为（面向使用者）

1) 查询类命令
- READ_TEMP：读取温度与状态信息。
- READ_PARAM：读取当前参数快照。

2) 设置类命令
- SET_TEMP=<value>
- SET_PID=<kp>,<ki>,<kd>
- SET_ALARM=<value>
- SET_SCHEDULE=<en>,<start_min>,<end_min>
- SET_LOG_PERIOD=<sec>

3) 配置类命令
- CONF:PID <kp>,<ki>,<kd>
- CONF:ALARM <value>
- CONF:SCH <en>,<start_min>,<end_min>
- CONF:LOGPERIOD <sec>

4) 日志类命令
- LOG_EXPORT：导出日志。
- LOG_CLEAR：清空日志。

5) 错误处理原则
- 命令格式错误：返回对应 BAD_xxx 错误。
- 参数越界：返回 OUT_OF_RANGE 类错误。
- 未知命令：返回 UNKNOWN_CMD。


2.4 运行与安全边界

1) 控制参数必须在定义范围内，超限输入不生效。
2) 协议解析必须“整体合法”才应用配置，不允许半合法输入写入参数。
3) 系统优先保证安全态，异常输入不应触发危险执行路径。


三、发布文件说明

1) WaterTempControl.bin
- 原始 Flash 镜像，常用于量产/脚本化烧录。

2) WaterTempControl.hex
- Intel HEX 镜像，兼容多种烧录工具。

3) WaterTempControl.axf
- 带调试符号镜像，用于断点调试、调用栈与符号定位。

4) WaterTempControl.map
- 链接映射文件，用于代码尺寸分析与符号分布检查。

5) SHA256SUMS.txt
- 发布文件哈希清单，用于完整性校验。


四、完整性校验方法

1) 使用 SHA-256 对发布文件计算哈希。
2) 与 SHA256SUMS.txt 中对应条目比对。
3) 若任一文件不一致，禁止进入烧录与交付流程。
