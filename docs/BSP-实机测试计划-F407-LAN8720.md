# BSP实机测试计划与用例（STM32F407 + LAN8720）

- 版本：v1.0
- 日期：2026-05-05
- 适用目标：`edgegateway-Debug`（STM32F407VGT6）
- 目的：覆盖BSP层关键硬件驱动的实机验证，形成可复现、可记录、可回归的测试基线。

## 1. 范围与原则

### 1.1 测试范围

- 显示驱动：LCD 8080
- 按键输入：SET/UP/DOWN
- 执行器：蜂鸣器/继电器
- 温度采样端口：DS18B20 x3
- 非易失存储：SPI Flash
- 有线网络：STM32F407 ETH MAC + LAN8720 PHY + RMII链路状态

### 1.2 执行原则

- BSP实机测试仅在物理板上执行，不以Host测试替代。
- 使用固件 `APP_HW_DRIVER_TEST_DISPLAY=1` 的硬件烟测流程执行。
- 每个测试项必须记录：输入、步骤、观察、结论、日志片段。

## 2. 环境准备

- 硬件：
  - STM32F407VGT6板卡
  - LAN8720 PHY与RJ45接口
  - DS18B20 3路
  - OLED/LCD、按键、蜂鸣器、继电器负载
- 上位机：
  - Keil或PlatformIO下载环境
  - 串口日志工具
  - 交换机/路由器（用于网口Link）
- 固件配置：
  - `edgegateway-Debug`
  - `APP_HW_DRIVER_TEST_DISPLAY=1`
  - `APP_USE_ETHERNET=1`

## 3. 测试入口

- 上电后进入 `HW SMOKE` 硬件烟测页面。
- 流程阶段：DISPLAY -> KEYS -> TEMP -> ACT -> SPIFLASH -> NETWORK -> DONE
- NETWORK阶段显示字段：
  - `R`：网络服务初始化状态（ready）
  - `L`：链路状态（link up/down）
  - `M`：链路速率（10/100）
  - `FD/HD`：全双工/半双工

## 4. 实机测试用例

### BSP-HW-001 显示驱动连贯性

- 目标：验证屏幕初始化、刷新、图形渲染无花屏/撕裂。
- 前置：上电进入DISPLAY阶段。
- 步骤：
  1. 观察纯色切换、移动方块、图元绘制。
  2. 持续观察5秒以上。
- 通过标准：
  - 无闪烁异常、无整屏错位、无明显残影。

### BSP-HW-002 按键扫描与状态迁移

- 目标：验证SET/UP/DOWN按键检测、边沿触发、阶段跳转。
- 前置：进入KEYS阶段。
- 步骤：
  1. 分别按SET/UP/DOWN。
  2. 检查屏幕显示与日志键值。
- 通过标准：
  - 三键可识别，状态按预期迁移。

### BSP-HW-003 温度端口可读性

- 目标：验证3路温度读取有效。
- 前置：进入TEMP阶段。
- 步骤：
  1. 观察A/B/C三路温度。
  2. 对任一路施加温变（手持加热/冷却）。
- 通过标准：
  - 至少一路实时变化，读数稳定无明显跳变失真。

### BSP-HW-004 执行器驱动

- 目标：验证蜂鸣器与继电器IO控制。
- 前置：进入ACT阶段。
- 步骤：
  1. 使用SET触发蜂鸣器切换。
  2. 使用UP触发继电器切换。
- 通过标准：
  - 蜂鸣器有明确声反馈，继电器动作与状态一致。

### BSP-HW-005 SPI Flash读写一致性

- 目标：验证SPI Flash写入与回读比对。
- 前置：进入SPIFLASH阶段。
- 步骤：
  1. 按SET执行写读校验。
  2. 观察地址、选择向量、RX预览。
- 通过标准：
  - wr=1, rd=1, cmp=1。

### BSP-HW-006 LAN8720链路状态测试

- 目标：验证LAN8720 PHY初始化、Link、速率/双工识别。
- 前置：进入NETWORK阶段。
- 步骤：
  1. 断开网线，观察 `L0`。
  2. 接入交换机网线，观察 `L1` 与速率字段。
  3. 按SET确认通过；若异常按DOWN标记失败。
- 通过标准：
  - 接线后 `L=1`，速率为10M或100M，双工字段可读。

### BSP-HW-007 LAN链路抖动恢复

- 目标：验证链路上下线检测与恢复。
- 前置：NETWORK阶段，已Link Up。
- 步骤：
  1. 拔网线，等待状态刷新。
  2. 重新插入，观察恢复。
- 通过标准：
  - 能从 `L1->L0->L1` 正确切换，无死锁。
