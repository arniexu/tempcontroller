# STM32F103 水温控制系统整体测试计划（V1.0）

- 项目名称：基于 STM32F103 的水温控制系统
- 适用版本：V1.8 设计基线
- 日期：2026-04-23
- 目标：覆盖上电自测、单元自测、压力测试、性能测试、集成功能测试，形成可执行闭环

---

## 1. 测试范围与目标

### 1.1 业务逻辑整合范围

- 采样链路：DS18B20 x3 -> 滤波/融合 -> 控制温度
- 控制链路：目标温度 -> PID -> 继电器时间比例控制
- 安全链路：超温/探头故障 -> 报警 -> 强制停热
- 调度链路：裸机时基调度（100ms/200ms/1s）
- HMI 链路：按键事件 -> 参数编辑 -> OLED 渲染
- 通信链路：串口查询/配置/PID 下发/日志导出
- 数据链路：运行日志记录 -> 导出 CSV
- 存储链路：参数更新 -> NV 双页 Flash 持久化

### 1.2 通过标准

- 严重缺陷（阻断安全或关键功能）为 0
- 关键功能用例通过率 100%
- 一般功能用例通过率 >= 95%
- 24h 压力运行无死机、无控制失稳
- Flash 预算与 NV 布局约束持续满足

---

## 2. 测试分层与执行策略

### 2.1 上电自测（POST）

目标：设备上电后 1~3 秒内完成关键组件健康检查，失败进入安全降级。

建议检查项：
- POST-01：参数区读取与 CRC 校验
- POST-02：温度传感器通道可读性（3 路至少 2 路可用）
- POST-03：RTC 可读性
- POST-04：继电器默认关闭状态
- POST-05：串口发送路径可用性（输出启动帧）
- POST-06：OLED 初始化与首帧渲染
- POST-07：按键扫描通道可读性

结果策略：
- 全部通过：进入正常业务流程
- 可降级故障：进入降级运行（禁止预约或限制加热）
- 不可恢复故障：进入报警态并保持加热关闭

### 2.2 单元自测（Host）

目标：模块级行为正确性与回归保护。

当前模块覆盖：
- scheduler
- pid_ctrl
- debug_log
- alarm_service
- param_store
- log_service
- heater_ctrl
- temp_manager
- schedule_service
- ui_service
- protocol_export
- app_main（smoke）

执行方式：
- 使用 firmware/tests/run_tests.ps1
- 每次代码提交必须通过

### 2.3 压力测试

目标：在长时、异常、频繁操作条件下验证稳定性。

场景：
- STRESS-01：连续运行 24h（采样+PID+日志+UI）
- STRESS-02：串口高频命令注入（查询/配置/导出混合）
- STRESS-03：按键高频连发与抖动注入
- STRESS-04：传感器间歇失效与恢复循环
- STRESS-05：频繁参数修改触发 NV 延迟写入

判定指标：
- 无死机/无看门狗复位（若启用）
- 无内存/状态机异常
- 控制输出无持续饱和异常

### 2.4 性能测试

目标：验证时序预算与资源占用满足低性能 MCU 约束。

指标：
- PERF-01：100ms 任务抖动
- PERF-02：200ms UI 刷新周期稳定性
- PERF-03：1s 控制任务执行时间与最坏路径
- PERF-04：串口导出吞吐与丢帧率
- PERF-05：Flash 占用（LR_1 <= 0xF800）
- PERF-06：RAM 峰值与日志缓存边界

### 2.5 集成功能测试

目标：按完整业务场景验证端到端功能。

场景清单：
- INT-01：正常升温到目标并稳定控温
- INT-02：超温触发报警并强制停热
- INT-03：预约到点启动与到点停止
- INT-04：掉电重启后参数恢复
- INT-05：上位机下发 PID 参数并实时生效
- INT-06：日志导出完整性与 CSV 可解析性
- INT-07：探头故障降级与恢复切换

## 3. 测试环境

- 硬件：STM32F103C8T6 控制板、3 路 DS18B20、继电器与加热负载模拟、OLED、按键
- 上位机：Windows + 串口工具/脚本
- 固件：当前主分支构建版本
- Host 自测：MinGW GCC（module_tests）
