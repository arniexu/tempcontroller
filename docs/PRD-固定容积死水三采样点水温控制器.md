# 产品需求规格说明书（PRD）

## 文档信息
- 产品名称：固定容积死水三采样点水温控制器
- 文档版本：V0.1（初始版）
- 日期：2026-05-01
- 适用范围：STM32F103 系列裸机控温固件与本地人机交互
- 目标用户：需要对固定水体进行安全、稳定、可追溯加热控制的用户

---

## 1. 产品目标与场景边界

### 1.1 目标
- 在固定容积静止水体中实现安全、稳定、可解释的闭环控温。
- 通过三采样点抑制分层带来的误判，降低体感忽冷忽热。
- 提供低学习成本的本地按键操作与清晰 UI 反馈。

### 1.2 场景边界
- 介质为固定容积、无循环泵、无主动搅拌的静止水体。
- 采样点为上层、中层、下层 3 点温度。
- 系统默认存在温度分层与热惯性，不按瞬时均匀混合建模。

### 1.3 非目标
- 不覆盖循环水系统、流动水系统、多加热源协同。
- 不覆盖云端远程控制与 App 联网（本版本以本地控制为主）。

---

## 2. 用户角色与核心诉求

### 2.1 典型角色
- 现场操作员：设置目标温度、观察状态、处理告警。
- 维护人员：调参、导出日志、定位异常。

### 2.2 核心诉求
- 明确知道当前是否安全，是否在加热，何时达到目标。
- 操作步骤少、按键逻辑统一、误操作可撤销。
- 异常时给出可执行提示，不只报码。

---

## 3. 功能总览
- 三点温度采集（T_top、T_mid、T_bot）与融合温度计算（T_fuse）。
- PID 闭环控制 + 继电器时间比例输出。
- 超温保护、探头故障保护、参数越界保护、上电自检保护。
- 本地 UI 显示运行态、温度、目标值、输出占空、告警信息。
- 三键交互（SET/UP/DOWN）支持查看、设置、确认、回退。
- 参数持久化（掉电记忆）与日志导出。

---

## 4. 用户操作流程

### 4.1 首次上电流程
1. 设备上电，进入 POST 自检（1~3 秒）。
2. UI 显示“自检中”。
3. 自检通过：进入主运行页。
4. 可降级故障：进入降级运行并提示“受限运行”。
5. 不可恢复故障：进入保护锁定，停热并报警。

### 4.2 日常控温流程
1. 用户在主运行页查看当前温度与目标温度。
2. 短按 SET 进入“目标温度编辑”。
3. 按 UP/DOWN 调整目标值，短按 SET 确认保存。
4. 返回主页后系统自动执行 PID 控温。
5. 达到稳态后维持目标附近温度，减少继电器抖动。

### 4.3 参数调优流程（维护）
1. 长按 SET 进入参数菜单。
2. 选择 PID 预设档位（快热/平衡/保温）或高级参数页。
3. 修改后确认并保存，系统立即生效并记录参数变更日志。
4. 用户可在“调参结果页”查看超调、稳定时间、当前波动。

### 4.4 异常处理流程
1. 系统检测到超温或关键故障，立刻停热并报警。
2. UI 显示异常类型、触发温度、建议动作。
3. 用户按 SET 查看详情，按 UP/DOWN 切换建议步骤。
4. 异常解除且满足恢复条件后，用户确认恢复运行。

---

## 5. UI 与按键交互细节

### 5.1 交互原则
- 页面层级不超过 3 级。
- 所有可编辑项统一“SET 进入/确认，UP 增加，DOWN 减少”。
- 页面底部固定显示按键提示，减少记忆负担。

### 5.2 页面定义
- P0 主运行页：
  - 显示 T_top/T_mid/T_bot/T_fuse、目标温度、输出占空、当前状态。
  - 显示加热图标、告警图标、预约图标（若启用）。
- P1 目标温度编辑页：
  - 显示当前值、步进值、允许范围、保存提示。
- P2 PID 快捷设置页：
  - 显示档位与说明：快热、平衡、保温。
- P3 PID 高级参数页：
  - 显示 Kp、Ki、Kd、积分限幅、输出限幅。
- P4 异常详情页：
  - 显示异常码、触发时间、触发源、恢复条件、建议动作。
- P5 日志导出页：
  - 显示导出状态、记录条数、串口连接提示。

### 5.3 按键事件定义
- SET 短按：进入下一级/确认当前项。
- SET 长按（>=1.2s）：从主页进入参数菜单；在编辑页执行“取消并返回”。
- UP 短按：数值 +1 步进或菜单上移。
- DOWN 短按：数值 -1 步进或菜单下移。
- UP/DOWN 长按（>=0.8s）：连续调整（加速步进）。
- 10 秒无操作：自动退出编辑页并回主页。

### 5.4 UI 刷新与一致性
- UI 刷新周期：200 ms。
- 温度主数据显示建议 0.1°C 精度，控制判断内部可用更高精度。
- 告警态颜色/符号必须显著区别于正常态，且持续可见直到解除。

## 6. 加热控制逻辑

### 6.1 三点融合与安全判据
- 采样输入：T_top、T_mid、T_bot。
- 默认融合温度：
  - T_fuse = 0.25*T_top + 0.50*T_mid + 0.25*T_bot。
- 安全温度：
  - T_safe = max(T_top, T_mid, T_bot)。
- 控制输入：默认使用 T_fuse。
- 安全保护：默认使用 T_safe。

### 6.2 运行状态
- IDLE：待机未加热。
- HEATING：PID 闭环控温。
- DEGRADED：降级运行（单探头异常）。
- ALARM：异常报警（停热）。
- LOCKED：保护锁定（需人工确认恢复）。

### 6.3 控制节拍
- 采样周期：1 s。
- 控制周期：1 s。
- 继电器窗口：10 s 时间比例控制。
- 继电器最小开/关保持时间：默认 2 s（可配置）。

### 6.4 控制策略要求
- 必须使用离散 PID，禁止退化成纯开关控制作为主策略。
- 必须具备 anti-windup（积分限幅/回算二选一，初版推荐积分限幅）。
- 输出限幅范围 0%~100%，并映射到时间比例窗口。
- 在分层显著时（T_top - T_bot 超阈值）降低积分作用或限制最大输出，避免顶部过热。

---

## 7. PID 参数的便捷定义

### 7.1 快捷档位（面向用户）
- D1 快热档：升温优先，允许小幅超调。
- D2 平衡档：速度与稳定性平衡（默认）。
- D3 保温档：波动最小，动作更平缓。

### 7.2 初始参数建议（可作为默认）
- D1 快热档：Kp=9.0, Ki=0.35, Kd=12.0。
- D2 平衡档：Kp=8.0, Ki=0.30, Kd=15.0。
- D3 保温档：Kp=6.5, Ki=0.22, Kd=18.0。

### 7.3 便捷调参规则
- 用户默认只选档位，不直接修改 Kp/Ki/Kd。
- 高级模式才开放单参数编辑，且每次调整幅度不超过 10%。
- 参数保存前执行边界校验：
  - Kp: 0.0~30.0
  - Ki: 0.0~5.0
  - Kd: 0.0~60.0
- 越界输入直接拒绝并提示“已恢复到可用区间”。

### 7.4 调参结果反馈
- 每轮参数修改后，系统在 UI 提供“调参观察窗口”（建议 5~10 分钟）。
- 输出 3 个用户可理解指标：
  - 达温时间
  - 最大超调
  - 稳态波动范围

## 8. 异常粗粒度分级与安全保护

### 8.1 粗粒度异常分级
- E1 提示级（可继续运行）：
  - 单次读数抖动、短时通信重试。
- E2 降级级（受限运行）：
  - 单探头连续异常、RTC 异常导致预约不可用。
- E3 保护级（立即停热）：
  - 超温、双探头及以上失效、关键参数损坏不可恢复。

### 8.2 安全保护动作矩阵
- 超温（T_safe > 阈值）：立刻停热 + 蜂鸣器 + UI 红色告警 + 事件记录。
- 单探头故障：进入 DEGRADED，限制输出上限并提示尽快检修。
- 双探头故障：进入 ALARM/LOCKED，停热并要求人工确认。
- 参数 CRC 错误：加载默认安全参数，标记“参数已恢复默认”。
- 上电自检失败：保持停热，禁止进入 HEATING。

### 8.3 恢复条件
- 超温恢复：温度低于恢复阈值并持续一段时间（例如 60 s）后，允许人工确认恢复。
- 探头故障恢复：连续 N 次读数有效后解除故障标记（N 默认 5）。
- 锁定状态恢复：必须由用户确认，不能静默自动恢复。

## 9. 参数与数据

### 9.1 持久化参数
- 目标温度
- PID 档位与高级参数
- 超温阈值与回差
- 采样周期/记录周期
- 预约参数（若启用）

### 9.2 日志要求
- 运行日志字段至少包括：时间戳、T_top、T_mid、T_bot、T_fuse、T_set、PID 输出、继电器状态、运行状态、告警码。
- 告警日志必须包含：触发时间、触发源、触发值、解除时间、处理动作。

---

## 10. 验收标准（初始版）

### 10.1 功能验收
- 用户可在 3 步内完成目标温度设置。
- 三点采样可持续更新并显示，异常时有明确提示。
- PID 控制可在设定时间内达到目标并维持稳定。
- 超温保护触发后 1 个控制周期内完成停热。

### 10.2 交互验收
- 主页面可直接看到当前状态、目标、输出、告警。
- 各页面按键行为一致，无冲突手势。
- 10 秒无操作自动回主页面。

### 10.3 安全验收
- E3 级异常必须强制停热。
- 异常恢复前必须满足恢复条件且有明确用户确认动作。
- 任何参数损坏都不能导致默认加热开启。
