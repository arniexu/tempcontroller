# STM32F103 水温控制系统设计文档（V1.0）

- 项目名称：基于 STM32F103 的水温控制系统
- 文档版本：V1.0
- 日期：2026-04-22
- 适用阶段：课程设计 / 毕业设计 / 小批量打样

---

## 1. 项目目标与范围

本项目设计一套基于 STM32F103 的水温控制设备，实现以下功能：

1. 基于定制 PCB 实现完整硬件系统，PCB 丝印包含用户 Logo。
2. 采用 3 路 DS18B20 温度采集。
3. 使用 PID 算法进行闭环控温（非简单开关控制）。
4. OLED 显示温度与时间。
5. 按键可设置目标温度与参数。
6. 继电器自动控制加热棒。
7. 超温报警（温度 > 60℃，蜂鸣器报警并切断加热）。
8. 断电保存目标温度与关键参数。
9. 支持预约加热（定时开始/结束）。
10. 温度数据可记录并导出到电脑。
11. 设计、焊接、调试、验收过程按步骤留存照片与视频。

---

## 2. 关键指标

- 控温范围：20℃~60℃（默认）
- 报警阈值：60℃（可配置）
- 采样周期：1 s
- 控制周期：1 s
- 显示刷新：200 ms
- 控温稳态误差：≤ ±0.5℃（合理保温结构 + PID 调参后）
- 数据记录周期：1 s / 5 s / 10 s（可选）

---

## 3. 系统总体方案

### 3.1 系统框图

电源模块（12V 输入）→ 5V/3.3V 供电 → STM32F103RCT6 主控

主控外设：
- DS18B20×3（温度采集）
- OLED（I2C 显示）
- 按键（参数设置）
- 继电器驱动（控制加热棒）
- 蜂鸣器（超温报警）
- RTC（时间与预约）
- EEPROM/Flash（掉电参数保存）
- UART-USB（数据导出）

### 3.2 控制逻辑

1. 周期读取 3 路温度并进行滤波。
2. 计算控制温度（默认三路平均，也可配置主探头优先）。
3. PID 计算输出控制量（0~100%）。
4. 按时间比例驱动继电器控制加热。
5. 执行超温保护、预约逻辑、数据显示与日志记录。

## 4. 硬件设计

### 4.1 主控

- MCU：STM32F103RCT6（72MHz，资源余量更充足，适合预约、日志记录、OLED 菜单与串口导出等完整功能集成）
- 封装建议：LQFP64，便于 GPIO 资源分配与后续功能扩展
- 选型理由：相比 STM32F103C8T6，RCT6 在 Flash、RAM 和 IO 资源上更宽裕，更适合本项目的 3 路测温、PID 控温、RTC、掉电存储、预约和数据导出组合需求

### 4.2 温度采集

- 传感器：DS18B20 × 3
- 建议连接：3 路独立 OneWire GPIO（抗干扰更好）
- 每路上拉：4.7kΩ 到 3.3V
- 探头建议使用防水不锈钢封装

### 4.3 显示与输入

- OLED：0.96 寸 128×64，SSD1306，I2C 接口
- 按键：SET、UP、DOWN、BACK（或确认）

### 4.4 执行与报警

- 加热执行：继电器控制加热棒
- 驱动：NPN/MOS 管 + 反向二极管（电磁继电器线圈）
- 蜂鸣器：有源蜂鸣器（GPIO 控制）

### 4.5 时间与存储

- RTC：DS3231（推荐，精度高）
- 参数存储：AT24C02/AT24C64 或 MCU 内部 Flash

### 4.6 通信导出

- UART 转 USB：CH340/CP2102
- 导出格式：CSV 文本

### 4.7 电源与安全

- 输入：12V DC
- 降压：12V→5V（Buck），5V→3.3V（LDO）
- 保护：反接保护、保险丝、TVS（建议）
- 强弱电分区：加热回路与主控区隔离布线

## 5. PCB 设计要求

1. 双层 PCB（常规即可）。
2. 顶层丝印加入用户 Logo（避开焊盘与测试点）。
3. 功率区与信号区分区布局，继电器及高压端子靠板边。
4. 关键器件近端去耦：0.1uF + 10uF。
5. 预留 SWD 下载口、串口调试口、测试点。
6. 继电器触点走线按电流留足线宽，必要时铺铜加厚。

## 6. 软件设计

### 6.1 软件架构

- 推荐软件栈：`STM32F10x StdPeriph + 裸机前后台架构`，不强制使用 RTOS
- 设计原则：模块解耦、定时调度、状态机驱动、参数集中管理、异常优先保护
- 分层结构：
  - BSP/驱动层：GPIO / TIM / I2C / UART / OneWire / RTC / EEPROM
  - 设备抽象层：ds18b20 / oled / relay / buzzer / key / rtc / storage / uart_export
  - 服务层：soft_timer / key_service / ui_service / log_service / alarm_service / schedule_service / param_service
  - 控制层：temp_acquire / temp_filter / pid_ctrl / heater_ctrl / safety_ctrl
  - 应用层：main_app / menu_app / export_app

### 6.3 任务模型

- 本项目推荐不用 RTOS，原因是任务规模有限、实时性要求明确、调试更简单
- 使用 `SysTick + 软件定时器 + 主循环状态机` 即可满足需求
- 任务调度采用“快中断、慢任务”原则：中断只置位标志，不做复杂业务处理

### 6.4 周期任务（建议）

- 1 ms：系统心跳、软定时器
- 100 ms：按键扫描、蜂鸣器状态机
- 200 ms：OLED 刷新
- 1 s：温度采样、PID 计算、继电器控制、日志记录、预约判断

建议主循环形式如下：

```c
while (1)
{
    scheduler_poll();
    key_service_process();
    ui_service_process();
    export_service_process();
    alarm_service_process();
}
```

### 6.5 模块职责划分

- `temp_manager`：读取 3 路 DS18B20，完成 CRC 检查、异常判定、滤波和控制温度计算
- `pid_ctrl`：根据目标温度和控制温度输出控制量，支持积分限幅与输出限幅
- `heater_ctrl`：将 PID 输出映射为时间比例控制窗口，驱动继电器并限制最小动作周期
- `alarm_service`：管理超温、探头故障、RTC 异常等告警，具备最高优先级
- `schedule_service`：根据 RTC 时间判断预约开始/结束，切换系统工作模式
- `param_store`：负责参数加载、CRC 校验、写入与默认值恢复
- `log_service`：定时记录温度、状态和报警事件，支持串口导出
- `ui_service`：负责 OLED 页面刷新、菜单切换、参数编辑与提示显示
- `protocol_export`：实现与电脑的数据交互协议，支持日志导出和参数查询

### 6.6 系统状态机

建议将系统抽象为以下运行状态：

- `IDLE`：待机，未加热，等待手动启动或预约到点
- `HEATING`：闭环控温运行中
- `SCHEDULED`：预约已启用，等待到达开始时间
- `ALARM`：超温或关键故障，强制停热并告警
- `EXPORT`：串口导出日志或调试数据

状态切换原则：

- `ALARM` 优先级最高，可从任意状态切入
- `SCHEDULED` 到点后进入 `HEATING`
- 预约结束、手动停止或故障恢复后返回 `IDLE`

### 6.7 温度处理

- 合法区间校验：-55℃~125℃
- 异常值剔除：CRC 错误/读数跳变过大
- 滤波建议：滑动平均（窗口 3~5）或中值滤波
- 控制温度建议：
  - 正常控制：`T_ctrl = (T1 + T2 + T3) / 3`
  - 安全判断：任一路超过阈值即触发报警，不仅依赖平均值
- 传感器故障策略：单路失效时提示降级运行；多路失效时禁止加热

### 6.8 PID 控温

离散位置式 PID：

$$
 u(k)=K_p e(k)+K_i\sum_{i=0}^{k} e(i)\Delta t+K_d\frac{e(k)-e(k-1)}{\Delta t}
$$

其中：

$$
 e(k)=T_{set}-T_{ctrl}
$$

- 输出限幅：$u(k)\in[0,100]$%
- 控制方式：时间比例控制（例如 10 s 窗口）
- 继电器保护：设置最小吸合/释放时间，减少抖动与寿命损耗
- 推荐控制窗口：`10 s`
- 推荐初始参数：`Kp=8.0, Ki=0.3, Kd=15.0`，后续依据水体容量与加热功率现场整定
- 必要保护：积分分离、积分限幅、输出限幅、防止小误差下继电器频繁抖动

### 6.9 超温保护

- 当任一路温度 > 60℃：
  - 立即关闭加热
  - 蜂鸣器报警
  - OLED 显示超温提示
  - 记录报警事件
- 当 DS18B20 连续多次读取失败时，也应进入故障保护或降级运行模式

### 6.10 掉电记忆

保存内容：
- 目标温度
- PID 参数
- 报警阈值
- 预约参数
- 记录周期

建议采用“结构体 + 版本号 + CRC”保存，防止异常断电导致参数损坏。
- 建议写入策略：只有参数变化且稳定数秒后再写入，避免频繁擦写 EEPROM/Flash

### 6.11 预约加热

- 可配置开始时间、结束时间
- 到点自动进入控温；到结束时间自动停止加热
- 页面显示预约状态（开/关、下一次执行时间）
- RTC 异常时禁止预约启动，并提示校时

### 6.12 数据记录与导出

- 本地环形日志记录：时间戳 + 三路温度 + 目标温度 + 输出 + 状态
- 串口导出命令示例：
  - `LOG_EXPORT`
  - `LOG_CLEAR`
  - `SET_TEMP xx.x`
- 导出 CSV 字段：
  - timestamp,t1,t2,t3,t_avg,set_temp,pid_out,heater,alarm

### 6.13 通信协议建议

- 物理层：UART，波特率 `115200`
- 帧格式建议：ASCII 文本命令，便于串口助手直接调试
- 推荐命令：
  - `READ_TEMP`
  - `READ_PARAM`
  - `SET_TEMP=45.0`
  - `SET_TIME=2026-04-22 20:30:00`
  - `LOG_EXPORT`
  - `LOG_CLEAR`
- 响应格式建议统一为：`OK,...` 或 `ERR,...`

### 6.14 软件栈结论

本项目的软件栈建议固定为：`STM32F103RCT6 + STM32F10x StdPeriph + 裸机定时调度 + 分层模块化设计 + 状态机控制`。该方案的优点是实现直接、调试成本低、足够支撑 3 路测温、PID 控温、预约、报警、掉电记忆和数据导出，适合作为课程设计、毕业设计和实际打样的第一版实现。

## 11. 风险与建议

- 若使用电磁继电器，高频控制会影响寿命，建议时间比例窗口不少于 5~10 s。
- 传感器线较长时需重点做抗干扰与屏蔽。
- 加热系统必须落实电气安全（绝缘、接地、保险、外壳防护）。
- 若后续希望更平滑控制，可升级为 SSR + 过零控制方案。
