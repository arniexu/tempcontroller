# STM32F103 水温控制系统设计文档（V2.4）

- 项目名称：基于 STM32F103 的水温控制系统
- 文档版本：V2.4
- 日期：2026-04-25
- 适用阶段：课程设计 / 毕业设计 / 小批量打样

---

## 1. 项目目标与范围

本项目设计一套基于 STM32F103 的水温控制设备，实现以下功能：

1. 基于定制 PCB 实现完整硬件系统，PCB 丝印包含用户 Logo。
2. 采用 3 路 DS18B20 温度采集。
3. 使用 PID 算法进行闭环控温（非简单开关控制）。
4. OLED 显示温度与时间。
5. 按键可设置目标温度与参数。
6. 继电器自动控制加热棒。
7. 超温报警（温度 > 60℃，蜂鸣器报警并切断加热）。
8. 断电保存目标温度与关键参数。
9. 支持预约加热（定时开始/结束）。
10. 温度数据可记录并导出到电脑。
11. 设计、焊接、调试、验收过程按步骤留存照片与视频。

---

## 2. 关键指标

- 控温范围：20℃~60℃（默认）
- 报警阈值：60℃（可配置）
- 采样周期：1 s
- 控制周期：1 s
- 显示刷新：200 ms
- 控温稳态误差：≤ ±0.5℃（合理保温结构 + PID 调参后）
- 数据记录周期：1 s / 5 s / 10 s（可选）

## 3. 系统总体方案

### 3.1 系统框图

电源模块（12V 输入）→ 5V/3.3V 供电 → STM32F103C8T6 主控

主控外设：
- DS18B20×3（温度采集）
- OLED（I2C 显示）
- 按键（参数设置）
- 继电器驱动（控制加热棒）
- 蜂鸣器（超温报警）
- RTC（时间与预约）
- EEPROM/Flash（掉电参数保存）
- UART-USB（数据导出）

### 3.2 控制逻辑

1. 周期读取 3 路温度并进行滤波。
2. 计算控制温度（默认三路平均，也可配置主探头优先）。
3. PID 计算输出控制量（0~100%）。
4. 按时间比例驱动继电器控制加热。
5. 执行超温保护、预约逻辑、数据显示与日志记录。

## 4. 硬件设计

### 4.1 主控

- MCU：STM32F103C8T6（72MHz，可满足本项目 3 路测温、PID 控温、预约、日志与串口导出功能需求）
- 封装建议：LQFP64，便于 GPIO 资源分配与后续功能扩展
- 选型理由：STM32F103C8T6 具备 72MHz 主频与足够的 GPIO/外设资源，能够覆盖本项目 3 路测温、PID 控温、RTC、掉电参数保存、预约与数据导出等核心需求，且成本更低、获取更方便。

### 4.2 温度采集

- 传感器：DS18B20 × 3
- 建议连接：3 路独立 OneWire GPIO（抗干扰更好）
- 每路上拉：4.7kΩ 到 3.3V
- 探头建议使用防水不锈钢封装
- 测点物理位置建议：
  - `T1`：靠近加热棒的下部区域，用于尽早感知局部升温
  - `T2`：水体中部，作为主要控制参考点
  - `T3`：上部或出水口附近，用于观察温度分层与超温风险
- 工程约定：若未单独切换主探头策略，则默认记录并显示 `T1/T2/T3` 三路数据，同时用 `T_ctrl` 作为控制温度参与 PID 运算

### 4.3 显示与输入

- OLED：0.96 寸 128×64，SSD1306，I2C 接口
- 按键：SET、UP、DOWN、BACK（或确认）
- 当前驱动接线约定：
  - `KEY_SET/UP/DOWN/BACK`：`PA0/PA1/PA2/PA3`，上拉输入，低电平按下
  - `OLED SSD1306`：`I2C2(PB10=SCL, PB11=SDA)`

### 4.4 执行与报警

- 加热执行：继电器控制加热棒
- 驱动：NPN/MOS 管 + 反向二极管（电磁继电器线圈）
- 蜂鸣器：有源蜂鸣器（GPIO 控制）
- 当前蜂鸣器驱动引脚约定：`PB13`（高电平鸣叫）

### 4.5 时间与存储

- RTC：DS3231（推荐，精度高）
- 参数存储：AT24C02/AT24C64 或 MCU 内部 Flash

### 4.6 通信导出

- UART 转 USB：CH340/CP2102
- 导出格式：CSV 文本

### 4.7 电源与安全

- 输入：12V DC
- 降压：12V→5V（Buck），5V→3.3V（LDO）
- 保护：反接保护、保险丝、TVS（建议）
- 强弱电分区：加热回路与主控区隔离布线

## 5. PCB 设计要求

1. 双层 PCB（常规即可）。
2. 顶层丝印加入用户 Logo（避开焊盘与测试点）。
3. 功率区与信号区分区布局，继电器及高压端子靠板边。
4. 关键器件近端去耦：0.1uF + 10uF。
5. 预留 SWD 下载口、串口调试口、测试点。
6. 继电器触点走线按电流留足线宽，必要时铺铜加厚。

## 6. 软件设计

### 6.1 软件架构

- 软件栈固定为：`STM32F10x StdPeriph + 裸机前后台架构`，禁止引入 RTOS
- 设计原则：模块解耦、定时调度、状态机驱动、参数集中管理、异常优先保护
- 分层结构：
  - BSP/驱动层：GPIO / TIM / I2C / UART / OneWire / RTC / EEPROM
  - 设备抽象层：ds18b20 / oled / relay / buzzer / key / rtc / storage / uart_export
  - 服务层：soft_timer / key_service / ui_service / log_service / alarm_service / schedule_service / param_service
  - 控制层：temp_acquire / temp_filter / pid_ctrl / heater_ctrl / safety_ctrl
  - 应用层：main_app / menu_app / export_app

### 6.3 任务模型

- 本项目必须不用 RTOS，原因是任务规模有限、实时性要求明确、调试更简单
- 使用 `SysTick + 软件定时器 + 主循环状态机` 即可满足需求
- 禁止复杂调度策略（如多级优先级抢占、动态任务管理、运行时任务拓扑变更）
- 任务调度采用“快中断、慢任务”原则：中断只置位标志，不做复杂业务处理，主循环按固定顺序轮询服务模块

### 6.4 周期任务（建议）

- 1 ms：系统心跳、软定时器
- 100 ms：按键扫描、蜂鸣器状态机
- 200 ms：OLED 刷新
- 1 s：温度采样、PID 计算、继电器控制、日志记录、预约判断

建议主循环形式如下：

```c
while (1)
{
    scheduler_poll();
    key_service_process();
    ui_service_process();
    export_service_process();
    alarm_service_process();
}
```

### 6.5 模块职责划分

- `temp_manager`：读取 3 路 DS18B20，完成 CRC 检查、异常判定、滤波和控制温度计算
- `pid_ctrl`：根据目标温度和控制温度输出控制量，支持积分限幅与输出限幅
- `heater_ctrl`：将 PID 输出映射为时间比例控制窗口，驱动继电器并限制最小动作周期
- `alarm_service`：管理超温、探头故障、RTC 异常等告警，具备最高优先级
- `schedule_service`：根据 RTC 时间判断预约开始/结束，切换系统工作模式
- `param_store`：负责参数加载、CRC 校验、写入与默认值恢复
- `log_service`：定时记录温度、状态和报警事件，支持串口导出
- `ui_service`：负责 OLED 页面刷新、菜单切换、参数编辑与提示显示
- `protocol_export`：实现与电脑的数据交互协议，支持日志导出和参数查询

### 6.5.1 设备驱动实现约束（BSP）

- 驱动层统一采用裸机轮询方式，不使用 RTOS 与中间件任务框架。
- 驱动实现保持简洁高效，避免动态内存与复杂调度逻辑。
- `bsp_ds18b20` 采用标准 OneWire 时序与 CRC 校验，支持独立 3 路采样与失败重试。
- `bsp_rtc` 采用标准备份域初始化流程，优先 LSE，失败回退 LSI，并使用备份寄存器标记初始化状态。
- `bsp_uart` 采用轮询收发，发送路径需具备超时保护，避免异常硬件状态导致主循环阻塞。
- `bsp_relay` 仅负责电平驱动与状态缓存，控制策略由上层 `heater_ctrl` 统一管理。

### 6.6 系统状态机

建议将系统抽象为以下运行状态：

- `IDLE`：待机，未加热，等待手动启动或预约到点
- `HEATING`：闭环控温运行中
- `SCHEDULED`：预约已启用，等待到达开始时间
- `ALARM`：超温或关键故障，强制停热并告警
- `EXPORT`：串口导出日志或调试数据

状态切换原则：

- `ALARM` 优先级最高，可从任意状态切入
- `SCHEDULED` 到点后进入 `HEATING`
- 预约结束、手动停止或故障恢复后返回 `IDLE`

### 6.7 温度处理

- 合法区间校验：-55℃~125℃
- 异常值剔除：CRC 错误/读数跳变过大
- 滤波建议：滑动平均（窗口 3~5）或中值滤波
- 控制温度建议：
  - 正常控制：`T_ctrl = (T1 + T2 + T3) / 3`
  - 安全判断：任一路超过阈值即触发报警，不仅依赖平均值
- 传感器故障策略：单路失效时提示降级运行；多路失效时禁止加热

### 6.8 PID 控温

离散位置式 PID：

$$
 u(k)=K_p e(k)+K_i\sum_{i=0}^{k} e(i)\Delta t+K_d\frac{e(k)-e(k-1)}{\Delta t}
$$

其中：

$$
 e(k)=T_{set}-T_{ctrl}
$$

- 输出限幅：$u(k)\in[0,100]$%
- 建议控制周期：`1 s`
- 说明：对水温这类慢热系统，`1 s` 控制周期通常足够；同时 DS18B20 在高分辨率下转换时间接近 `750 ms`，因此不建议在初版实现中盲目缩短控制周期
- 控制方式：时间比例控制（例如 10 s 窗口）
- 继电器保护：设置最小吸合/释放时间，减少抖动与寿命损耗
- 推荐控制窗口：`10 s`
- 推荐初始参数：`Kp=8.0, Ki=0.3, Kd=15.0`，后续依据水体容量与加热功率现场整定
- 必要保护：积分分离、积分限幅、输出限幅、防止小误差下继电器频繁抖动

PID 现场调试建议：

- 第一步仅开启比例项：先令 `Ki=0, Kd=0`，逐步增大 `Kp`，观察升温速度与振荡趋势
- 第二步加入积分项：在 `Kp` 基本稳定后缓慢增大 `Ki`，用于消除稳态误差
- 第三步少量加入微分项：若过冲明显，再增加少量 `Kd` 抑制温度变化过快导致的超调
- 建议每次只修改一个参数，单次调整幅度不超过 `10%`
- 每轮调试建议记录：超调量、整定时间、稳态误差、继电器动作频率

### 6.9 超温保护

- 当任一路温度 > 60℃：
  - 立即关闭加热
  - 蜂鸣器报警
  - OLED 显示超温提示
  - 记录报警事件
- 当 DS18B20 连续多次读取失败时，也应进入故障保护或降级运行模式

### 6.10 掉电记忆

保存内容：
- 目标温度
- PID 参数
- 报警阈值
- 预约参数
- 记录周期

NV 变量存储采用“外部 EEPROM 优先、片内 Flash 兜底”的双后端方案：

- 主方案：AT24Cxx 外部 EEPROM（默认 AT24C02，I2C 地址 `0x50`）
  - 存储布局：双槽轮换（SlotA/SlotB），每槽一份完整参数记录
  - 写入策略：参数变更后延迟 3~5 s 再写；当前实现固定为 3 s 延迟刷写，每次写入较旧槽并通过 `seq` 选最新
- 兜底方案：STM32F103C8T6 片内 Flash 双页轮换（当未接 EEPROM 或项目配置禁用 EEPROM 时）
  - 布局：保留末尾 2 页作为 NV 区（双页轮换）
  - 页大小：1 KB
  - 地址规划（64 KB Flash 版本）：
    - `PageA`: `0x0800F800` ~ `0x0800FBFF`
    - `PageB`: `0x0800FC00` ~ `0x0800FFFF`
- 记录格式：`header + payload + crc32`
  - `header`: `magic(4) + layout_ver(2) + seq(4) + payload_len(2)`
  - `payload`: 参数结构体（目标温度、报警阈值、PID、预约、记录周期）
  - `crc32`: 对 `header(不含crc字段) + payload` 计算
- 生效规则：启动时扫描两页，选择 `seq` 最大且 CRC 正确的记录作为有效参数
- Flash 写入策略：每次写入新页并在成功后擦除旧页，避免半写入损坏
- 降级策略：若全部存储槽均无有效记录，则回退默认参数并写入一份初始记录

实现约束：
- 不使用文件系统、不使用复杂 KV 组件，仅采用定长结构体日志方案
- 写 Flash 期间短临界区保护，完成后立即恢复主循环

### 6.10.1 Flash Layout 与容量约束

- 设备总 Flash 按 `64 KB` 进行容量预算（即使个别芯片存在超容也不作为设计依据）。
- 应用程序区：`0x08000000` ~ `0x0800F7FF`（62 KB）。
- NV 参数区（保留，不可放置代码/常量）：`0x0800F800` ~ `0x0800FFFF`（2 KB）。
- 链接产物约束：`Load Region LR_1` 大小必须 `<= 0xF800`。
- 发布约束：每次发布需附带 map 文件容量摘要（LR_1、ER_RO、ER_RW、ER_ZI）。
- 工程约束：禁止通过修改链接地址占用 NV 区；如需扩展容量，必须先更新设计文档并通过评审。

### 6.11 预约加热

- 可配置开始时间、结束时间
- 到点自动进入控温；到结束时间自动停止加热
- 页面显示预约状态（开/关、下一次执行时间）
- RTC 异常时禁止预约启动，并提示校时

### 6.12 数据记录与导出

- 本地环形日志记录：时间戳 + 三路温度 + 目标温度 + 输出 + 状态
- 串口支持两类用途：
  - 历史日志导出：用于调试后回放与保存 CSV
  - 实时数据上报：用于电脑端绘制温度/PID/继电器状态曲线
- 导出 CSV 字段：
  - timestamp,t1,t2,t3,t_avg,set_temp,pid_out,heater,alarm

实时绘图数据建议字段：

- `seq`：递增帧序号，用于检测漏帧
- `time_ms`：设备本地毫秒计数，用于恢复真实时间轴
- `t1,t2,t3,t_ctrl,set_temp,pid_out,heater,alarm,mode`

推荐实时数据帧示例：

```text
DATA:105,105000,24.81,24.76,24.79,24.79,45.00,62.50,1,0,1
```

其中字段顺序为：

```text
DATA:seq,time_ms,t1,t2,t3,t_ctrl,set_temp,pid_out,heater,alarm,mode
```

### 6.13 通信协议建议

- 物理层：UART，波特率 `115200`
- 协议层建议：采用基于 UART 的 SCPI 风格 ASCII 文本协议，便于串口助手直接调试和电脑端快速接入绘图脚本
- 工作模式建议同时支持：
  - 主机轮询模式：上位机定时发送查询命令，设备返回一帧数据
  - 流式上报模式：上位机发送 `STREAM:ON` 后，设备按控制周期主动输出 `DATA:` 帧
- 推荐命令：
  - `MEAS:TEMP?`
  - `MEAS:ALL?`
  - `READ:PID?`
  - `READ:STATE?`
  - `CONF:SETP 45.0`
  - `CONF:PID 8.0,0.3,15.0`
  - `LOG:EXPORT?`
  - `LOG:CLEAR`
  - `STREAM:ON`
  - `STREAM:OFF`
- 当前固件已支持的关键命令：
  - `READ_TEMP`
  - `READ_PARAM`
  - `SET_TEMP=<float>`
  - `SET_PID=<kp>,<ki>,<kd>`
  - `CONF:PID <kp>,<ki>,<kd>`
  - `SET_ALARM=<float>`
  - `CONF:ALARM <float>`
  - `SET_SCHEDULE=<en>,<start_min>,<end_min>`
  - `CONF:SCH <en>,<start_min>,<end_min>`
  - `SET_LOG_PERIOD=<sec>`
  - `CONF:LOGPERIOD <sec>`
  - `LOG_CLEAR`
  - `LOG_EXPORT`
- PID 下发约束：
  - `kp` 范围：`0.0~30.0`
  - `ki` 范围：`0.0~5.0`
  - `kd` 范围：`0.0~50.0`
  - 成功响应：`OK,PID_APPLIED`
  - 越界响应：`ERR,PID_OUT_OF_RANGE`
- 报警阈值下发约束：
  - 范围：`40.0~90.0`
  - 成功响应：`OK,ALARM_APPLIED`
  - 越界响应：`ERR,ALARM_OUT_OF_RANGE`
- 预约参数下发约束：
  - `en`：`0/1`
  - `start_min/end_min`：`0~1439`
  - 成功响应：`OK,SCHEDULE_APPLIED`
  - 越界响应：`ERR,SCHEDULE_OUT_OF_RANGE`
- 日志周期下发约束：
  - 范围：`1~60` 秒
  - 成功响应：`OK,LOG_PERIOD_APPLIED`
  - 越界响应：`ERR,LOG_PERIOD_OUT_OF_RANGE`
- 响应格式建议：
  - 查询命令返回单行 ASCII 数据
  - 设置命令统一返回 `OK` 或 `ERR,...`
  - 流式模式下仅输出 `DATA:` 开头的数据帧，避免与调试日志混发造成解析复杂化
- 轮询绘图示例：

```text
主机发: MEAS:ALL?
设备回: 25.31,25.44,25.28,25.34,45.00,36.50,1,0,1
```

- 流式绘图示例：

```text
主机发: STREAM:ON
设备回: OK
设备持续发: DATA:seq,time_ms,t1,t2,t3,t_ctrl,set_temp,pid_out,heater,alarm,mode
```

### 6.14 软件栈结论

本项目的软件栈建议固定为：`STM32F103C8T6 + STM32F10x StdPeriph + 裸机定时调度 + 分层模块化设计 + 状态机控制`。该方案的优点是实现直接、调试成本低、足够支撑 3 路测温、PID 控温、预约、报警、掉电记忆和数据导出，适合作为课程设计、毕业设计和实际打样的第一版实现。

## 7. 人机界面设计

### 7.1 主界面

- 时间（RTC）
- T1/T2/T3
- 平均温度
- 目标温度
- 加热状态
- 预约状态
- 报警状态

### 7.2 菜单结构

1. 目标温度设置
2. PID 参数设置
3. 报警阈值设置
4. 预约加热设置
5. 数据导出
6. 系统信息/恢复默认

### 7.3 交互逻辑（参考业界嵌入式实现）

- 交互范式采用“页面状态机 + 按键事件驱动 + 参数即时预览”的轻量模式。
- 事件模型：`SET/UP/DOWN/BACK` 短按、`SET` 长按、`UP/DOWN` 连发。
- 运行原则：
  - 非编辑态：`UP/DOWN` 切页，`SET` 进入编辑，`BACK` 回主页。
  - 编辑态：`UP/DOWN` 调整参数，`SET` 确认（PID 页用于字段切换），`BACK` 退出编辑。
  - 长按 `SET`：
    - 编辑态下快速退出编辑并返回非编辑态；
    - `INFO` 页面非编辑态下执行恢复默认参数。
- 页面定义：
  - `HOME`：运行总览与状态提示。
  - `SET_TEMP`：目标温度调整。
  - `PID`：`Kp/Ki/Kd` 字段轮转编辑。
  - `ALARM`：超温阈值调整。
  - `SCHEDULE/EXPORT/INFO`：预约、导出与信息页面。

### 7.4 参数编辑约束

- 目标温度：`20.0~60.0℃`，步进 `0.5℃`。
- 报警阈值：`40.0~90.0℃`，步进 `0.5℃`。
- `Kp`：`0.0~30.0`，步进 `0.2`。
- `Ki`：`0.0~5.0`，步进 `0.05`。
- `Kd`：`0.0~50.0`，步进 `0.5`。
- 所有参数编辑必须执行边界钳位，防止越界输入导致控制异常。
- 编辑参数变更后需走统一 NV 延迟写入策略，避免高频擦写 Flash。

### 7.5 UI 驱动接入实现

- 按键接入：`ui_service_tick_100ms` 周期扫描按键状态，完成短按/长按/连发事件转换。
- 事件处理：使用小型环形队列串行处理按键事件，避免同周期多键抖动导致状态机错乱。
- OLED 渲染：`ui_service_tick_200ms` 生成页面文本并调用 `bsp_oled_draw_text` 与 `bsp_oled_refresh` 刷新。
- 页面渲染分层：
  - 第 1 行：模式与页面标识
  - 第 2 行：控制温度与目标温度
  - 第 3 行：PID 输出、加热与报警状态
  - 第 4 行：当前页面的参数摘要
- `EXPORT` 页面实现约束：
  - 显示当前日志条数与日志周期（秒）
  - 显示导出命令提示（`LOG_EXPORT`）
  - 不在 UI 线程直接触发导出，仅提供状态与操作提示
- `INFO` 页面实现约束：
  - 长按 `SET` 执行恢复默认参数操作（目标温度、报警阈值、PID、预约参数、记录周期）
  - 恢复后显示短时反馈提示，实际持久化由统一 NV 延迟写入路径完成
- 约束：渲染逻辑仅显示状态，不直接改变控制策略参数，参数修改只通过按键事件路径完成。

## 8. 测试与验收

本章节采用“分层测试 + 业务链路追踪”的整体策略，完整计划见：

- 传感器断线、短路异常测试
- 上电默认加热关闭
- 异常复位后恢复安全状态

### 8.3 分层测试要求

- 上电自测：覆盖参数区、传感器、RTC、继电器默认态、串口与显示初始化状态。

- 压力测试：要求包含 24h 稳定运行、按键高频操作、串口高频通信与故障注入。
- 性能测试：要求采集任务周期抖动、关键路径耗时、RAM/Flash 占用与导出吞吐。
- 集成功能测试：要求覆盖温控闭环、报警保护、预约控制、掉电恢复、数据导出与上位机参数下发。

### 8.4 测试通过判据

- 安全相关用例必须 100% 通过。
- 关键业务链路用例必须 100% 通过。
- 一般用例通过率不低于 95%。
- 不允许存在阻断级缺陷进入发布候选。

## 11. 风险与建议

- 若使用电磁继电器，高频控制会影响寿命，建议时间比例窗口不少于 5~10 s。
- 传感器线较长时需重点做抗干扰与屏蔽。
- 加热系统必须落实电气安全（绝缘、接地、保险、外壳防护）。
- 若后续希望更平滑控制，可升级为 SSR + 过零控制方案。
