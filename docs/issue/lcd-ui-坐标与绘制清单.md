# LCD界面坐标与绘制清单（240x320）

- 适用设计稿：docs/issue/lcd-ui-preview.html
- 坐标系：左上角为原点 (0,0)，x 向右增大，y 向下增大
- 可视区：仅 .screen 区域，宽 240，高 320
- 单位：像素

---

## 1. 通用绘制规则

1. 固定顺序：背景 -> 分隔线/卡片 -> 文字与数值 -> 状态色强调 -> 角落符号提示（可选）。
2. 正常运行态页面禁止常驻文字按键条（如“设定/上调/下调”）。
3. 告警色仅用于高风险字段（超温值、停热状态、故障项）。
4. 一级信息（主温度、目标值、告警状态）字号高于二级信息至少 1 档。
5. 若需要提示按键，仅允许使用符号：`◀` `▶` `OK` `↵` `⤒` `⤓`。
6. 每页最多 2 个符号提示，放置在角落区域，且不得干扰主数值。
7. 编辑页或帮助页可一次性显示完整按键说明；普通运行页不显示“按XX键”文字。

## 2. 通用组件坐标

### 2.1 头部组件（壳页）
- 标题：x=24, y=24
- 副标题：x=26, y=56
- 头部分隔线：x=24, y=82, w=192, h=2

### 2.2 主值卡片（value-card）
- 默认：x=20, y=106, w=200, h=110
- 矮卡（tall）：x=20, y=106, w=200, h=88

### 2.3 符号提示位（hint-symbol）
- 默认位置：x=188, y=304（右下角）
- 推荐字号：13 px
- 最大字符数：5（例如：`◀ ▶`、`OK ⤒`）
- 约束：每页至多 2 个符号，不得覆盖关键状态与主温度

### 2.4 详情键值框（kv-box）
- 容器：x=20, y=102, w=200, h=112（4行）或 h=134（5行）
- 单行高度建议：24~26

## 4. 固件实现映射建议

1. 建议建立页面枚举：启动页、首页、设定温度、参数档位、运行详情、超温报警、探头异常、自检结果、导出页。
2. 建议建立组件绘制函数：
- draw_header(title, subtitle)
- draw_value_card(label, value)
- draw_kv_box(items)
- draw_hint_symbol(symbol_text)
3. 状态色映射：
- 正常：绿色
- 运行中：青色或暖色
- 告警：红色
4. 文案建议统一放入常量表，避免硬编码分散。

---

## 5. 验收检查表

- 普通运行页是否已移除常驻文字按键条。
- 每页符号提示是否不超过2个且位于角落。
- 首页是否可一眼看到目标温度与安全最高温。
- 超温页是否同时显示恢复条件与锁定状态。
- 详情页数值表达是否统一（百分比、秒、温度）。
- 页面切换时关键文案是否保持一致语义。
